# v1.0.6

## 修复内容

修复开机启动时因Windows网络栈未稳定导致Portal探测过早执行、自动登录失败的问题。

### 核心改进

- **系统启动宽限期**：系统启动后90秒内等待网络稳定，避免在DHCP/路由/Clash TUN初始化阶段误判。
- **网络就绪检测**：Portal探测前确认物理网卡UP、已获私有IPv4、默认路由存在、Portal TCP可连。
- **连续稳定检测**：网络就绪需连续5秒稳定后才开始Portal探测。
- **虚拟网卡IP过滤**：198.18.x.x（Clash TUN）、198.19.x.x、169.254.x.x、127.x.x.x均被识别为虚拟/无效源地址，不会被绑定为出站接口。

### 和 v1.0.5 相比

- v1.0.5 启动后直接探测Portal，如果Windows网络未就绪会失败
- v1.0.6 启动后先等待网络稳定，确认物理网卡和路由就绪后再探测
- v1.0.6 修复了Clash TUN虚拟网卡IP被错误绑定为源地址的问题

## 升级方式

- 替换旧版exe即可
- 已初始化用户无需重新`--init`
